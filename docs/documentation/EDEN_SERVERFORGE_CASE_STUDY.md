# EDEN and ServerForge Competition Case Study

## Executive summary

PROMETHEUS was implemented and hardened against a real local repository inside EDEN, the local execution environment. Codex accelerated implementation, integration and debugging. PROMETHEUS governed the Command-to-Proof mission. ServerForge preserved a sanitized operational narrative. GitHub Pages published the judge-facing interface, machine-readable proof receipts, Capability Genome and documentation.

This case study proves the engineering path without disclosing proprietary internals.

## Public architecture

```text
EDEN local execution body
  → Codex implementation and hardening
  → PROMETHEUS Command-to-Proof
  → ServerForge sanitized observability
  → GitHub repository and Pages deployment
  → judge-facing proof surface
```

### EDEN

EDEN is the local execution body. For this competition it provided the Windows-based workspace, repository access, Python runtime, command shell, local files and development tools required to build and verify the executable package.

The public case study intentionally does not publish machine-specific paths, credentials, private state, private prompts or unrelated local systems.

### Codex

Codex worked against the real competition repository and accelerated:

- CLI implementation
- deterministic mission execution
- replay-failure reproduction
- receipt and hash verification
- tests and contest checks
- public interface construction
- deployment repair
- control wiring
- competition hardening
- documentation and submission preparation

The human creator retained product definition, architecture, proof semantics, visual direction, IP boundaries and release authority.

### PROMETHEUS

PROMETHEUS converted the mission into the canonical sequence:

```text
PROMETHEUS
→ AIS-Ω
→ Adversarial Twin
→ SECA
→ HYDRA
→ ProofGrid
→ Capability Genome
→ Build Truth
```

The mission reproduced a duplicate consequential operation, denied unsupported promotion, changed replay behavior through an idempotent guard, passed five tests, verified eight artifacts, sealed a ProofGrid receipt, preserved the repair as a Capability Genome and reused it on a related task.

### ServerForge

ServerForge served as the observability transport and case-study recorder. Its public role is limited to sanitized operational events such as:

- mission admission
- agent handoffs
- bounded failure
- SECA denial
- HYDRA repair
- proof verification
- recovery state
- evidence publication
- final promotion

ServerForge does not expose credentials, Discord secrets, local machine state, private prompts or proprietary routing logic.

### Public node

The public GitHub Pages node remains available when EDEN is offline. It presents a deterministic visualization of the verified snapshot and provides direct access to receipts, hashes, Capability Genome data and judge documentation.
